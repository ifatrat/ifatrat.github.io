ftp传输模式

发表于 2016-04-05

这几天在调试设备上的程序，因为需要把Linux上制作的固件上传到Windows主机上，然后通过设备上的程序下载Windows上的固件来升级。问题来了，设备上的程序总是提示校验失败，经过一番排查，发现是linux下的ftp工具造成的。

FTP传输模式

Linux上的FTP命令工具下有Ascii和Binary两种模式，默认是Ascii模式。这两种模式在不同的系统之间传输文件的时候，会造成很大哦不同. 我们看下man的说明:

  • ascii Set the file transfer type to network ASCII. This is the default type.
  • binary Set the file transfer type to support binary image transfer.

使用FileZilla等工具，可以选择自动模式，工具软件会根据需要来决定传输模式

Ascii

ASCII模式将回车换行转换为本机的回车字符，比如：UNIX下是\n,Windows下是\r\n，Mac下是\r。
ASCII模式下会转换文件，不能说是不同系统对回车换行解释不同，而是不同的系统有不同的行结束符。UNIX系统下行结束符是一个字节，即十六进制的0A，而Windows的系统是两个字节，即十六进制的0D0A，所以当你用ASCII方式从UNIX的FTP Server下载文件到Windows系统上时(不管是二进制或者文本文件)，每检测到一个字节是0A，就会自动插入一个0D，所以如果你的文件是二进制文件，比如可执行文件、压缩包什么的，就肯定不能用了。如果你的文件就是UNIX下的文本文件，你用ASCII模式是正确的，要是误用了Binary模式，你在Windows上看这个文件是没有换行的，里面是一个个的黑方块

Binary

Binary模式下，ftp工具不对传输内容做任何改动，所以当你传输非文本二进制的时候，最好使用该模式
